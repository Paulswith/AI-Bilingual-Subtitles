# Feature Specification: 翻译性能与 UI 增强

**Feature Branch**: `001-perf-cache-ui`
**Created**: 2026-03-03
**Status**: Draft
**Input**: 首次批量翻译速度控制在 5 秒内；有缓存应该提示已加载到缓存；目标源语言暂且支持中文，其它无需支持；如果秘钥已经配置，需要在 UI 上提示已配置

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 缓存加载提示 (Priority: P1)

用户打开已有翻译缓存的视频页面时，系统自动检测并提示"已从缓存加载"，让用户知道无需等待翻译。

**Why this priority**: 缓存是减少用户等待的核心功能，明确的提示能避免用户疑惑"翻译是否在工作"，提升用户信任感。

**Independent Test**: 打开之前访问过的视频页面，检查是否在字幕加载后显示缓存提示，且不触发新的翻译请求。

**Acceptance Scenarios**:

1. **Given** 用户之前访问过此视频且翻译已缓存，**When** 用户再次打开同一视频，**Then** 字幕区域显示"已从缓存加载"提示，且不发起新的翻译请求
2. **Given** 视频字幕内容已变更（哈希不匹配），**When** 用户打开视频，**Then** 不显示缓存提示，自动重新翻译
3. **Given** 用户首次访问此视频，**When** 打开视频，**Then** 不显示缓存提示，正常显示翻译进度

---

### User Story 2 - 翻译性能优化 (Priority: P1)

首次批量翻译时，系统必须在 5 秒内完成首批字幕的翻译并显示，让用户快速看到结果。

**Why this priority**: 5 秒是用户耐心的关键阈值，超过此时长用户会认为系统卡顿或失效，直接影响用户对扩展的第一印象。

**Independent Test**: 在首次访问视频时，从开始翻译到第一批字幕显示的时间不超过 5 秒。

**Acceptance Scenarios**:

1. **Given** 用户首次访问视频且有网络连接，**When** 触发翻译功能，**Then** 首批字幕在 5 秒内显示
2. **Given** 使用 Google 翻译服务，**When** 批量翻译 100 条字幕，**Then** 首批（前 20 条）在 5 秒内显示，其余继续加载
3. **Given** 使用 OpenAI 服务且 API 配置正确，**When** 触发翻译，**Then** 首批字幕在 5 秒内显示

---

### User Story 3 - API Key 配置状态提示 (Priority: P2)

用户在设置中配置了 OpenAI API Key 后，扩展弹窗的 UI 上清晰显示"API Key 已配置"状态，让用户知道高级功能可用。

**Why this priority**: 用户配置 API Key 后常不确定是否生效，明确的状态提示减少用户反复进入设置页面确认的操作。

**Independent Test**: 配置 API Key 后，扩展弹窗显示"已配置"状态；清除 API Key 后，提示消失。

**Acceptance Scenarios**:

1. **Given** 用户已配置 OpenAI API Key，**When** 打开扩展弹窗，**Then** 在服务选择区域显示"✓ API Key 已配置"提示
2. **Given** 用户未配置 API Key，**When** 打开扩展弹窗，**Then** 不显示"已配置"提示，显示"未配置"或无提示
3. **Given** 用户清除了 API Key，**When** 打开扩展弹窗，**Then** "已配置"提示消失

---

### User Story 4 - 源语言限制 (Priority: P3)

系统当前仅支持将**英文字幕**翻译为中文，其他语言字幕不触发翻译流程或提示不支持。

**Why this priority**: 专注核心用户群体（观看英文视频学习中文的用户），避免资源浪费在小语种场景，同时简化产品逻辑。

**Independent Test**: 检测视频字幕语言，英文字幕时正常翻译，其他语言时不启动翻译或显示不支持提示。

**Acceptance Scenarios**:

1. **Given** 视频字幕语言为英文，**When** 触发翻译功能，**Then** 正常执行翻译流程（翻译为中文）
2. **Given** 视频字幕语言为中文字幕，**When** 触发翻译功能，**Then** 不启动翻译，显示"暂不支持此语言"提示
3. **Given** 视频字幕语言为日文、韩文等其他语言，**When** 触发翻译功能，**Then** 不启动翻译

---

### Edge Cases

- 网络不稳定时，首批翻译超时如何处理（显示错误提示，支持重试）
- 缓存损坏或格式错误时如何处理（自动清除损坏缓存，重新翻译）
- API Key 无效时的错误提示（明确告知"API Key 无效，请检查配置"）
- 字幕文件过大（超过 1000 条）时的分批处理策略
- 视频中途切换音轨/字幕轨道时的缓存校验

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统 MUST 在检测到字幕后检查本地缓存，若缓存存在且有效则显示"已从缓存加载"提示
- **FR-002**: 系统 MUST 在 5 秒内完成首批字幕（至少前 20 条）的翻译并渲染显示
- **FR-003**: 系统 MUST 在扩展弹窗 UI 上显示 API Key 配置状态（已配置/未配置）
- **FR-004**: 系统 MUST 检测字幕源语言，仅对英文源字幕 trigger 翻译流程（翻译为中文）
- **FR-005**: 系统 MUST 对非英文字幕不启动翻译，显示"暂不支持此语言（仅支持英文）"提示
- **FR-006**: 缓存键 MUST 基于字幕内容哈希，字幕内容变更时自动失效
- **FR-007**: 翻译进度 MUST 持久化存储，页面刷新后可恢复并继续
- **FR-008**: API Key 配置状态 MUST 实时同步到 UI，配置变更后立即反映

### Key Entities

- **翻译缓存**: 存储字幕原文、翻译结果、内容哈希、时间戳
- **配置状态**: API Key 是否已配置、当前选择的翻译服务、批处理参数
- **字幕轨道**: 字幕语言、字幕内容哈希、当前激活状态

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 首次批量翻译时，90% 的情况下首批字幕（前 20 条）在 5 秒内显示
- **SC-002**: 缓存命中时，100% 的情况下显示"已从缓存加载"提示
- **SC-003**: API Key 已配置时，100% 的情况下 UI 显示"已配置"状态
- **SC-004**: 英文字幕检测准确率达到 95% 以上
- **SC-005**: 非英文字幕 100% 不触发翻译流程
