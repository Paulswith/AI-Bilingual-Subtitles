## Entity 1: TranslationCache (翻译缓存)

**Purpose**: 存储已翻译的字幕内容，避免重复翻译

### Fields

| Field | Type | Description |
|-------|------|-------------|
| videoId | string | 视频唯一标识 (从 URL 提取) |
| subtitleHash | string | 字幕内容哈希 (SHA-256 截取 16 字符) |
| originalSubs | Array\<SubtitleCue\> | 原始字幕数组 |
| translatedSubs | Array\<SubtitleCue\> | 翻译后字幕数组 |
| createdAt | number | 缓存创建时间戳 |
| expiresAt | number | 缓存过期时间戳 (createdAt + 30 天) |
| accessCount | number | 访问次数 (用于缓存清理策略) |

### SubtitleCue Structure

```javascript
{
  id: number,           // 字幕序号
  startTime: number,    // 开始时间 (秒)
  endTime: number,      // 结束时间 (秒)
  text: string,         // 字幕文本
  translatedText?: string // 翻译后文本 (可选)
}
```

### Validation Rules

- subtitleHash 必须基于 originalSubs 全文内容计算
- expiresAt 必须大于 createdAt
- translatedSubs 长度必须等于 originalSubs 长度

### State Transitions

```
[新建缓存] → (翻译完成) → [有效缓存] → (30 天) → [过期缓存]
                              ↓
                        (字幕变更) → [失效缓存]
```

## Entity 2: ConfigState (配置状态)

**Purpose**: 管理用户配置及 API Key 状态

### Fields

| Field | Type | Description |
|-------|------|-------------|
| service | 'google' \| 'openai' | 翻译服务提供商 |
| apiKey | string? | OpenAI API Key (加密存储，可选) |
| apiBaseUrl | string? | OpenAI 兼容接口地址 (可选) |
| model | string? | 模型名称 (如 gpt-3.5-turbo) |
| batchSize | number | 每批翻译字幕数量 (默认 10) |
| requestInterval | number | 批次间请求间隔毫秒 (默认 200) |
| debugMode | boolean | 调试日志开关 (默认 false) |
| lastConfiguredAt | number? | 最后配置时间戳 |

### Computed Properties

```javascript
get isApiKeyConfigured(): boolean {
  return !!this.apiKey && this.apiKey.length > 0;
}

get shouldUseOpenAI(): boolean {
  return this.service === 'openai' && this.isApiKeyConfigured;
}
```

## Entity 3: SubtitleTrack (字幕轨道)

**Purpose**: 表示当前视频的字幕轨道信息

### Fields

| Field | Type | Description |
|-------|------|-------------|
| language | string | 语言代码 (zh, en, ja 等) |
| label | string | 字幕标签 (如"中文"、"English") |
| isAutoGenerated | boolean | 是否自动生成字幕 |
| contentHash | string | 字幕内容哈希 |
| cues | Array\<SubtitleCue\> | 字幕片段数组 |
| detectedLanguage | 'zh' \| 'en' \| 'ja' \| 'ko' \| 'unknown' | 检测到的语言 |

### Language Detection

```javascript
function detectLanguage(cues): 'zh' | 'en' | 'ja' | 'ko' | 'unknown' {
  const sampleText = cues.slice(0, 10).map(c => c.text).join(' ');

  if (/[\u4e00-\u9fa5]/.test(sampleText)) return 'zh';
  if (/[\u3040-\u309f\u30a0-\u30ff]/.test(sampleText)) return 'ja';
  if (/[\uac00-\ud7af]/.test(sampleText)) return 'ko';
  if (/[a-zA-Z]/.test(sampleText)) return 'en';

  return 'unknown';
}
```

## Entity 4: TranslationProgress (翻译进度)

**Purpose**: 持久化翻译进度，支持页面刷新后恢复

### Fields

| Field | Type | Description |
|-------|------|-------------|
| videoId | string | 视频 ID |

### TranslationError Structure

```javascript

```

## Storage Strategy

| Entity | Storage Type | Persistence | Sync |
|--------|--------------|-------------|------|
| TranslationCache | storage.local | 30 天 | 不同步 |
| ConfigState | storage.sync | 永久 | 跨设备同步 |
| SubtitleTrack | memory | 页面生命周期 | 不同步 |
| TranslationProgress | storage.local | 7 天 | 不同步 |

**Rationale**:
- 缓存数据体积大，使用 storage.local (容量优先)
- 配置数据小且需要跨设备，使用 storage.sync
- 字幕轨道临时使用，内存存储即可
- 进度需要持久化但短期有效

## Cache Eviction Policy

**LRU + TTL 混合策略**:

1. **TTL**: 超过 30 天的缓存标记为过期
2. **LRU**: 存储容量达到 80% 时，清理最少访问的缓存
3. **手动清理**: 用户点击"清除缓存"时全部清空

```javascript
async function evictOldCaches() {
  const allCaches = await getAllCaches();
  const now = Date.now();

  // 1. 清理过期
  const expired = allCaches.filter(c => c.expiresAt < now);
  await deleteCaches(expired.map(c => c.videoId));

  // 2. LRU (如果仍超限)
  const remaining = await getAllCaches();
  if (remaining.length > MAX_CACHE_COUNT) {
    remaining.sort((a, b) => a.accessCount - b.accessCount);
    const toDelete = remaining.slice(0, remaining.length - MAX_CACHE_COUNT);
    await deleteCaches(toDelete.map(c => c.videoId));
  }
}
```
