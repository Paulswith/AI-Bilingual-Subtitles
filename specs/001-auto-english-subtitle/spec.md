# Feature Specification: 英文字幕自动识别与拒绝提示

**Feature Branch**: `001-auto-english-subtitle`  
**Created**: 2026-03-03  
**Status**: Draft  
**Input**: User description: "能否自动找到英文的字幕进行翻译，如果找不到的时候，提示用户拒绝生成;"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 自动识别英文字幕并翻译 (Priority: P1)

用户进入视频页面后，系统自动识别当前可用字幕中是否存在英文字幕；如果存在，系统自动选择英文字幕并进入翻译流程。

**Why this priority**: 这是核心价值路径，直接决定用户是否能“一步开始翻译”。

**Independent Test**: 在同时存在多种字幕（含英文）的视频页面，系统自动选中英文字幕并开始翻译，无需用户手动切换。

**Acceptance Scenarios**:

1. **Given** 页面存在英文字幕轨道，**When** 系统开始字幕检测，**Then** 系统自动识别并使用英文字幕作为翻译输入。
2. **Given** 页面存在多条字幕轨道（中/英/其他），**When** 自动检测完成，**Then** 系统优先选择英文字幕并开始翻译。
3. **Given** 英文字幕已被识别并开始翻译，**When** 用户查看状态信息，**Then** 用户可看到“正在基于英文字幕翻译”或等效提示。

---

### User Story 2 - 找不到英文字幕时拒绝生成并提示 (Priority: P1)

当页面没有英文字幕时，系统不应继续生成翻译内容，而应明确提示用户“拒绝生成”的原因和下一步操作建议。

**Why this priority**: 防止错误输出和误导用户，确保系统行为可预期。

**Independent Test**: 在仅有非英文字幕或无字幕页面触发翻译，系统停止翻译并显示拒绝提示，不生成新翻译结果。

**Acceptance Scenarios**:

1. **Given** 页面没有英文字幕，**When** 用户触发翻译，**Then** 系统拒绝生成翻译并显示明确提示。
2. **Given** 页面仅有中文字幕或其他语言字幕，**When** 系统执行自动检测，**Then** 系统不进入翻译流程并给出引导提示（如切换至英文字幕）。
3. **Given** 系统已判定“拒绝生成”，**When** 用户重复触发翻译，**Then** 系统保持拒绝并重复可理解提示，直到字幕条件满足。

---

### User Story 3 - 拒绝状态可恢复 (Priority: P2)

用户在看到“拒绝生成”后，如果后续切换到英文字幕，系统应能自动或手动重新进入翻译流程。

**Why this priority**: 保证功能闭环，避免用户卡在拒绝状态无法继续。

**Independent Test**: 先在无英文字幕页面触发拒绝，再切换到英文字幕，系统可成功开始翻译。

**Acceptance Scenarios**:

1. **Given** 当前为拒绝生成状态，**When** 用户切换到英文字幕并重新触发翻译，**Then** 系统成功开始翻译并清除拒绝提示。
2. **Given** 当前为拒绝生成状态，**When** 页面字幕轨道更新为包含英文字幕，**Then** 系统在下一次检测时可识别并允许翻译。

### Edge Cases

- 页面字幕轨道延迟加载，首次检测未找到英文字幕，后续加载出英文字幕。
- 英文字幕标签存在但内容为空或不可读取。
- 页面字幕轨道频繁切换，系统需避免“拒绝/翻译”状态抖动。
- 用户在翻译中途切到非英文字幕，系统需停止新增翻译并给出说明。
- 页面根本无字幕轨道，系统需直接拒绝生成并提示原因。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统 MUST 自动检测当前页面可用字幕轨道，并识别是否存在英文字幕。
- **FR-002**: 系统 MUST 在存在英文字幕时优先使用英文字幕作为翻译输入。
- **FR-003**: 系统 MUST 在找不到英文字幕时拒绝生成翻译结果，不得继续发起翻译输出。
- **FR-004**: 系统 MUST 在拒绝生成时向用户展示明确原因和下一步建议（例如“请切换到英文字幕后重试”）。
- **FR-005**: 用户 MUST 能在拒绝状态下重新触发检测；当英文字幕可用时系统必须允许进入翻译流程。
- **FR-006**: 系统 MUST 在字幕轨道变化后更新检测结果，避免使用过期的字幕判定。
- **FR-007**: 系统 MUST 在日志或状态中记录“已识别英文字幕”与“拒绝生成”两类结果，便于用户理解当前状态。

### Key Entities *(include if feature involves data)*

- **Subtitle Track（字幕轨道）**: 表示页面中的单条字幕来源，关键属性包括语言标识、可用状态、内容可读状态。
- **Translation Eligibility（翻译资格）**: 表示当前是否允许生成翻译的判定结果，关键属性包括是否可翻译、拒绝原因、更新时间。
- **Rejection Notice（拒绝提示）**: 表示向用户展示的拒绝生成反馈，关键属性包括提示文本、建议动作、展示状态。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 在存在英文字幕的页面中，95% 以上的会话可在触发翻译后 2 秒内完成英文字幕识别并进入翻译流程。
- **SC-002**: 在无英文字幕的页面中，100% 的翻译触发请求都会被拒绝生成，并展示可理解的拒绝提示。
- **SC-003**: 在“先拒绝后切换英文字幕”的场景中，90% 以上会话可在一次重试内成功进入翻译流程。
- **SC-004**: 用户对“为何未生成翻译”的理解正确率达到 95% 以上（通过可用性测试或问卷验证）。

## Assumptions

- 英文字幕是当前唯一允许触发翻译生成的源语言。
- 用户可通过页面播放器切换字幕轨道。
- 当页面未提供字幕或字幕不可读取时，系统按“找不到英文字幕”处理并拒绝生成。
