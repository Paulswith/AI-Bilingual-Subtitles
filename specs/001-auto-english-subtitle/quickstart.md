# Quickstart: 英文字幕自动识别与拒绝提示

## 目标

快速验证以下能力：
- 自动识别英文字幕并进入翻译
- 无英文字幕时拒绝生成并提示
- 从拒绝状态恢复到可翻译状态

## 前置条件

1. 已加载扩展开发版本。
2. 打开带视频字幕的网站页面（优先可切换多语言字幕）。
3. 扩展配置保持默认（Google 翻译可用）。

## 场景 A：存在英文字幕

1. 打开一个提供英文字幕的视频页面。
2. 保持或切换到任意字幕状态，触发翻译（自动或手动）。
3. 预期结果：
   - 系统自动识别英文字幕并开始翻译。
   - 状态区显示等效文案：正在基于英文字幕翻译。

## 场景 B：不存在英文字幕（拒绝生成）

1. 打开仅有中文字幕或无字幕的视频页面。
2. 触发翻译。
3. 预期结果：
   - 系统拒绝生成翻译结果。
   - 提示文案包含原因 + 建议动作（例如：请切换到英文字幕后重试）。

## 场景 C：拒绝后恢复

1. 在场景 B 的页面保持“拒绝生成”状态。
2. 切换到英文字幕轨道。
3. 再次触发翻译（或等待自动重判定）。
4. 预期结果：
   - 系统进入可翻译状态并开始翻译。
   - 拒绝提示被清除。

## 场景 D：轨道延迟加载

1. 打开字幕轨道晚于视频出现的页面。
2. 首次触发翻译可能进入 `pending` 或拒绝状态。
3. 等字幕轨道加载后重试。
4. 预期结果：
   - 系统可在后续检测中识别英文轨道并恢复翻译能力。

## 验证记录建议

- 记录每个场景的：
  - 页面 URL（可匿名）
  - 首次触发时间
  - 判定状态（eligible/rejected/pending）
  - 是否符合预期
