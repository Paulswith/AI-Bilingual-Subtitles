# Quickstart: 翻译模块重构 — 逐条翻译与实时双语展示

## 目标

快速验证以下核心能力：
- 逐条翻译并实时展示双语字幕
- 翻译进度实时更新
- 单条失败不阻塞后续翻译
- 缓存命中时直接加载

## 前置条件

1. 已加载扩展开发版本。
2. 打开带英文字幕的视频页面。
3. 扩展配置保持默认（Google 翻译可用）。

## 场景 A：逐条翻译与实时展示

1. 打开英文字幕视频页面。
2. 等待扩展自动检测字幕。
3. 触发翻译（自动或手动点击"翻译字幕"）。
4. 预期结果：
   - 视频播放到第一条字幕时，该条已显示"中文 + 英文"双行。
   - 控制面板显示"已翻译 1/N 条"，数字逐条递增。
   - 播放到尚未翻译完的字幕时，仅显示英文原文。
   - 翻译完成后自动变为双语。

## 场景 B：进度面板验证

1. 在场景 A 翻译进行中，查看控制面板。
2. 预期结果：
   - 显示"已翻译 N/M 条"，N 随时间递增。
   - 全部完成后显示"翻译完成"。
   - 弹窗中的进度与控制面板一致。

## 场景 C：单条失败降级

1. 模拟翻译失败（如断网后恢复）。
2. 预期结果：
   - 失败条目跳过，后续字幕继续翻译。
   - 控制面板显示失败条数信息。
   - 再次触发翻译时，仅重试失败条目。

## 场景 D：缓存命中

1. 在场景 A 翻译完成后，关闭页面。
2. 重新打开同一视频页面。
3. 预期结果：
   - 字幕立即以双语形式展示（无翻译请求）。
   - 控制面板显示"已从缓存加载"。

## 场景 E：翻译中断

1. 翻译进行到一半时，切换到另一个视频或刷新页面。
2. 重新打开原视频。
3. 预期结果：
   - 不会看到之前的不完整翻译。
   - 重新触发翻译从头开始。

## 验证记录建议

- 记录每个场景的：
  - 首条翻译可见时间（触发翻译到看到双语字幕的耗时）
  - 翻译速度（每条约耗时多少秒）
  - 进度面板数字是否与实际一致
  - 失败条目是否正确跳过
  - 缓存加载是否即时
